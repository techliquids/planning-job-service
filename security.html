🔐 Security & Compliance Framework

Job Boy implements enterprise-grade security across authentication, data protection, payment processing, and regulatory compliance. Security is not an afterthought but a core architectural principle, embedded at every layer of the platform.

🛡️ Security Pillars

We follow a defense-in-depth approach across 5 critical areas: Authentication, Data Protection, Payment Security, Fraud Prevention, and Regulatory Compliance.

🔑 Authentication & Access Control

User Authentication

  • Mobile OTP via SMS (6-digit codes, 10-minute expiry)
  • Email verification with unique tokens
  • Google OAuth 2.0 for social login
  • Session tokens with 24-hour expiration
  • Automatic logout on inactivity (30 minutes)
  • Password hashing with bcrypt (12 rounds)

Access Control

  • Role-based access control (Employer, Worker, Admin)
  • JWT token-based authorization
  • Two-factor authentication (2FA) for sensitive operations
  • Device fingerprinting for multi-device detection
  • Rate limiting to prevent brute force attacks
  • API key management for integrations

Authentication Flow

Secure Login Process

1
User enters phone number - Input validation on client & server
2
OTP generated & sent via SMS - Expires in 10 minutes, single use
3
User submits OTP - Server validates against database
4
JWT token issued - Contains user ID, role, permissions
5
Token stored securely - In secure storage, not localStorage

🔒 Data Protection & Encryption

Encryption Standards

  • TLS 1.3 for all API communications
  • AES-256 encryption for data at rest
  • Database encryption with AWS KMS
  • Field-level encryption for sensitive data (phone, SSN)
  • End-to-end encryption for chat messages
  • HTTPS only (no HTTP traffic)

Data Classification & Handling

🟠 Sensitive Data

Phone, Email, SSN, Bank Account, Payment info

Protection: Field-level encryption, tokenization, masked in logs

🟣 PII Data

Name, Address, Photos, Work history

Protection: Encrypted at rest, access logs, audit trail

Data Residency & Privacy

All Indian customer data stored in AWS Mumbai region
GDPR compliance for EU users (data processed in EU)
Data deletion requests honored within 30 days
Regular backup (daily), tested recovery procedures
Data access logs maintained for 1 year
Anonymization of data for analytics

💳 Payment & Financial Security

PCI DSS Compliance

  • Level 1 PCI DSS Certification (highest security standard)
  • No direct credit card storage - Tokenization via Razorpay
  • Payment gateway: Razorpay (RBI-approved)
  • Secure payment forms with CSP headers
  • 3D Secure authentication for card payments
  • Quarterly security audits & penetration testing

Transaction Security

Secure Payment Flow

1
Payment initiated - Employer selects amount & method
2
Fraud checks - AI system analyzes transaction for red flags
3
Amount held in escrow - Platform wallet, not withdrawn yet
4
Job completion - Worker finishes, submits proof
5
Payment released - Funds transferred to worker wallet
6
Worker withdrawal - Can withdraw to bank (next business day)

💰 Wallet Security

Worker wallets are held in a secure escrow account with ICICI Bank. All transactions are logged. Funds cannot be lost due to platform failure. Insurance coverage for gross negligence.

🚨 Fraud Detection & Prevention

Threat Model

💳 Payment Fraud

Stolen card usage, chargebacks, fake transactions

Mitigation: 3D Secure, velocity checks, device fingerprinting

👤 Account Takeover

Unauthorized access to user accounts

Mitigation: 2FA, login alerts, unusual activity detection

📱 SIM Swap Attacks

Attacker gains phone number for OTP interception

Mitigation: Email verification, unusual login alerts, 2FA

🤖 Bot Networks

Automated account creation, fake jobs, spam

Mitigation: CAPTCHA, rate limiting, behavioral analysis

📍 GPS Spoofing

Workers fake location during job execution

Mitigation: Location consistency checks, photo verification

⭐ Fake Reviews

Manipulated ratings to boost worker profile

Mitigation: Verified reviews, photo verification, NLP analysis

AI Fraud Detection System

Anomaly Detection: Isolation Forest algorithm detects unusual user behavior
Device Fingerprinting: Tracks device identifiers to detect identical twins
Velocity Checks: Flags rapid transactions from same account/device
Behavioral Biometrics: Analyzes typing patterns, scroll speed, click patterns
Network Analysis: Detects coordinated fraud rings with shared contacts/IPs
Reputation Scoring: Machine learning score (0-100) for each transaction
